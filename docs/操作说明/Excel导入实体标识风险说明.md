# Excel 导入实体标识风险说明

## 适用场景

用户在导入 Excel 或从 Excel 创建数据表时，看到类似提示：

```text
当前实体标识字段存在重复值
当前标识字段存在风险
实体标识存在重复值，请重新选择标识字段
```

这类提示表示：系统认为当前选择的实体标识字段，可能不能稳定、唯一地识别每一条业务实体。

## 什么是实体标识

在 ChronoTrace 中，一条业务对象会被系统保存为一个 `Entity`。

系统内部使用：

```text
数据表 + 实体编号
```

来判断“这是不是同一个业务实体”。

这里的实体编号来自当前数据表配置的实体标识字段，也就是 `identity_field_key`。

例如，资产表把“资产编号”设置为实体标识：

```text
资产编号    名称      状态
A001       笔记本    在用
A002       显示器    闲置
```

那么系统会用 `A001`、`A002` 来判断两行分别代表两个不同资产。

## 为什么会提示风险

实体标识字段必须尽量满足两个条件：

```text
唯一：同一张表里不能有两个不同实体使用同一个标识。
稳定：下次导入同一个实体时，仍然能拿到同一个标识。
```

如果实体标识选错，系统可能会发生误判：

```text
两个不同的人因为姓名相同，被当成同一个实体。
同一个资产因为编号变化，被当成一个新实体。
下个月重新导入时，因为没有携带上次生成的编码，历史被切断。
```

所以导入预览阶段会检查当前标识字段是否可靠。

## 重复值是最硬的风险

如果本次 Excel 中，当前实体标识字段出现重复值，系统会提示风险或错误。

例如：

```text
姓名    部门      社保基数
张三    财务部    5000
张三    销售部    6500
```

如果把“姓名”设置为实体标识，系统只能看到两个 `张三`，无法确认这是同一个人还是两个不同的人。

此时系统会提示：

```text
当前实体标识字段“姓名”存在重复值
```

并在预览中列出重复值、出现次数和所在行号。

这不是普通格式错误，而是实体匹配错误。继续导入可能导致不同实体被合并。

## 哪些字段不适合作为实体标识

以下字段通常不适合作为实体标识：

- 姓名
- 名称
- 部门
- 岗位
- 职位
- 状态
- 类型
- 性别
- 备注
- 长文本说明
- 枚举字段
- 布尔字段

原因是这些字段容易重复，或者会随时间变化。

例如：

```text
姓名会重名，也可能改名。
部门会调动。
岗位会变化。
状态本来就是会变化的数据。
类型通常很多行相同。
```

## 哪些字段适合作为实体标识

优先选择稳定编号类字段：

- 资产编号
- 员工号
- 工号
- 人员编码
- 供应商编码
- 合同号
- 订单号
- 证照编号
- 社保系统人员编号
- 业务系统中的唯一 ID

例如人员表建议优先使用：

```text
员工号 / 工号 / 人员编码
```

资产表建议优先使用：

```text
资产编号 / 资产号
```

供应商表建议优先使用：

```text
供应商编码
```

## 空值和唯一率偏低也会触发风险

从 Excel 创建新表时，系统会根据预览数据评估候选标识字段。

如果当前字段存在空值，会提示风险：

```text
存在空值
```

如果当前字段唯一率低，也会提示风险：

```text
唯一率偏低
```

唯一率可以简单理解为：

```text
不同值数量 / 非空行数量
```

例如 100 行数据里，某列只有 5 个不同值，这列就不适合作为实体标识。

## 系统生成编码的注意事项

有些导入场景会使用系统生成编码，例如 `person_code`、自动实体编码或 `system_code`。

这类编码也有使用边界。

如果系统提示：

```text
后续周期导入需继续携带该字段
```

意思是：本次生成的编码必须在后续 Excel 中继续保留。

否则下次导入同一个人或同一个资产时，系统无法知道它就是上次那条实体，可能会创建成新实体。

如果系统提示：

```text
system_code 按导入行号生成，只适合一次性新表接入
```

意思是：它只适合一次性导入历史数据，不适合每月、每周重复导入的周期表。因为行号很容易变化。

## 怎么处理这个提示

看到实体标识风险时，优先按下面顺序处理：

1. 检查预览中列出的重复值和行号。
2. 判断当前标识字段是不是姓名、名称、部门、状态等不稳定字段。
3. 如果有员工号、资产编号、供应商编码等稳定编号，改选稳定编号字段。
4. 如果单个字段无法唯一识别，可以先在 Excel 中新增一个组合标识列。
5. 如果使用系统生成编码，确认后续导入能持续携带同一个编码。
6. 重新生成预览，确认重复值风险消失后再提交导入。

## 组合标识示例

如果单个字段不唯一，可以在 Excel 中提前生成组合标识列。

例如员工号在不同公司可能重复：

```text
公司编码    员工号    姓名
BJ         001      张三
SH         001      李四
```

可以新增一列：

```text
人员唯一标识
BJ-001
SH-001
```

然后把“人员唯一标识”设置为实体标识。

## 快速判断

可以按下面的规则判断：

```text
如果这个字段会重复，不要作为实体标识。
如果这个字段会变化，不要作为实体标识。
如果下次导入拿不到同一个值，不要作为实体标识。
如果这个字段能长期稳定唯一识别业务对象，才适合作为实体标识。
```

常见正确选择：

```text
人员表：员工号、工号、人员编码。
资产表：资产编号、资产号。
供应商表：供应商编码。
合同表：合同号。
项目表：项目编号。
```

常见错误选择：

```text
姓名、名称、部门、岗位、状态、类型、备注、行号。
```
