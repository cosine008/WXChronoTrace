# Excel 导入缺失策略说明

## 适用场景

用户在导入 Excel 时，看到“缺失策略”选项，不确定应该选择“缺失保留”还是“缺失终止”。

## 什么是缺失记录

“缺失”不是指 Excel 中某个单元格为空，也不是某个字段没有填写。

在 Excel 导入中，“缺失记录”指的是：

```text
系统当前已有这条实体记录，但本次导入的 Excel 文件里没有出现这条实体记录。
```

例如，系统当前已有 3 条员工记录：

```text
A001 张三
A002 李四
A003 王五
```

本次导入的 Excel 只有：

```text
A001 张三
A002 李四
```

那么 `A003 王五` 就是缺失记录。因为它在系统当前数据中存在，但没有出现在本次 Excel 文件中。

## 缺失保留

选择“缺失保留”时，Excel 中没有出现的旧记录会继续保留，不会生成终止变更。

以上面的例子为例，导入后结果是：

```text
A001 按 Excel 内容新增、更新或保持不变
A002 按 Excel 内容新增、更新或保持不变
A003 王五继续有效
```

适合选择“缺失保留”的场景：

- 本次 Excel 只是局部数据。
- Excel 只包含某个部门、某个范围或某批需要处理的数据。
- Excel 不是完整台账。
- 不希望因为 Excel 少了某些行，就让系统里的旧记录失效。

## 缺失终止

选择“缺失终止”时，Excel 中没有出现的旧记录会生成“终止”变更。

以上面的例子为例，导入后结果是：

```text
A001 按 Excel 内容新增、更新或保持不变
A002 按 Excel 内容新增、更新或保持不变
A003 王五从本次导入的生效日期起终止
```

这里的“终止”通常不是物理删除历史数据，而是让这条记录从指定生效日期开始不再作为当前有效记录。

适合选择“缺失终止”的场景：

- 本次 Excel 是完整名单。
- 本次 Excel 是完整台账。
- 本次 Excel 代表某个日期下的全量有效数据快照。
- 如果旧记录没有出现在 Excel 中，就表示它已经离职、下架、退场、失效或不再有效。

## 快速判断

可以按下面的规则判断：

```text
如果本次 Excel 是局部数据，选择“缺失保留”。
如果本次 Excel 是完整全量数据，选择“缺失终止”。
```

## 第一次导入时的影响

如果是第一次通过 Excel 创建新表，系统里还没有旧记录，通常不会产生缺失记录。

因此，第一次导入时，“缺失策略”通常影响不大。

后续再次向同一张表导入 Excel 时，缺失策略就很重要。它决定了“系统中已有、但本次 Excel 没出现的旧记录”是继续保留，还是从生效日期开始终止。

## 连续型表和周期型表的默认倾向

当前系统中：

- 连续型表更常使用“缺失保留”。
- 周期型表更常使用“缺失终止”。

原因是连续型表经常处理局部变更，而周期型表更常导入某个周期的完整快照。

但最终仍应以本次 Excel 的业务含义为准：

- 如果 Excel 是局部补录或局部更新，选择“缺失保留”。
- 如果 Excel 是全量名单或完整快照，选择“缺失终止”。

## 示例

### 局部更新示例

本次 Excel 只包含“研发部”的员工，不包含其他部门员工。

这种情况下，应选择：

```text
缺失保留
```

否则，其他部门员工可能会因为没有出现在本次 Excel 中而被终止。

### 全量名单示例

本次 Excel 是“截至 2026-05-22 的全部在职员工名单”。

这种情况下，可以选择：

```text
缺失终止
```

如果某个旧员工没有出现在这份完整名单中，系统会认为该员工从本次导入生效日期起不再有效。

## 推荐话术

如果不确定应该选哪个，可以先向数据提供方确认：

```text
这份 Excel 是全量名单，还是只包含本次需要新增或修改的部分数据？
如果是部分数据，我会选择缺失保留。
如果是全量数据，我会选择缺失终止。
```
