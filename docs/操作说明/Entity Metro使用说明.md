# Entity Metro 使用说明

适用对象:业务用户、表管理员、审计/复核人员。

## 用途

Entity Metro 用来查看单个实体在时间轴上的演变过程。它把同一个实体的多个历史版本整理成一条可浏览的线路,适合回答下面几类问题:

- 这个实体什么时候创建、变更、终止。
- 哪些版本是关键站点,哪些只是普通过站版本。
- 某个字段在某次差异附近发生了什么变化。
- 当前账号在权限范围内能看到的字段快照是什么。

Entity Metro 仍然使用现有实体时间线数据,不会绕过 Current View 或 Diff Studio 已有的权限、脱敏和隐藏字段规则。

## 入口一:Current View

适用场景:从当前视图里快速查看某个实体的生命周期。

使用方式:

1. 打开目标表的 Current View。
2. 在主表中点击某个实体。
3. 右侧会打开“实体 Metro”抽屉。

Current View 入口特点:

- 使用抽屉形态,适合边看主表边看时间线。
- 默认展示 Metro 摘要和关键站点。
- 继续保留实体导出和标签历史入口。
- 关闭抽屉后,Current View 的搜索、排序、日期和当前选中态应保持不变。

## 入口二:Diff Studio Evidence Drawer

适用场景:在差异分析中继续深挖某个实体的前后演变。

使用方式:

1. 打开 Diff Studio。
2. 选中差异行,打开 Evidence Drawer。
3. 点击“打开实体 Metro”。
4. 系统会进入全屏 Entity Metro 页面。

Diff Studio 入口特点:

- 使用全屏形态,便于连续查看多个站点详情。
- 会带上差异上下文,包括字段 key、模式、相关 `record_id` 和返回链路。
- 返回时优先回到原来的 Diff Studio 页面;如果没有原始返回参数,回到该表的 Diff Studio 默认页。

## 关键站点 vs 全部版本

Entity Metro 提供两种浏览方式:

- `关键站点`:默认模式。只突出创建、终止、状态变化、部门变化、关键字段变化等重要版本,适合快速看脉络。
- `全部版本`:显示所有历史版本。非关键版本会作为小站点出现,适合排查“变化很细但不是关键事件”的情况。

建议先看关键站点,只有在需要核对细节时再切到全部版本。

## 字段高亮与 record_id 高亮

从 Diff Studio 进入时,Entity Metro 会尽量保留上下文高亮:

- 字段高亮:当前差异关联的字段会在站点详情里优先体现。
- `record_id` 高亮:如果 Diff Studio 能定位到对应版本,Metro 会把该版本作为高亮站点或优先选中站点。

从 Current View 进入时,默认是通用生命周期视角,通常不会带字段级高亮或指定 `record_id` 高亮。

## 权限与脱敏预期

Entity Metro 的展示遵循当前登录账号的权限边界:

- 无权查看的表或实体,不会因为 Metro 入口而额外暴露数据。
- 敏感字段如果在当前角色下应脱敏,Metro 中仍会显示脱敏结果,例如 `***` 或掩码值。
- 隐藏字段 / system 字段不会因为打开 Metro 而额外出现。
- 同一个实体在 Current View、Diff Studio、Entity Metro 中应保持一致的权限语义。

如果管理员发现 Metro 中的字段展示和 Current View / Diff Studio 不一致,应按“排查清单”处理并上报。

## P1 限制

当前 P1 版本有这些限制:

- 只支持单实体查看,不支持跨实体对比。
- Metro 内不提供编辑、审批、下载、导出或批量操作。
- 站点分类由前端基于现有 timeline records 计算,后端没有新增专用 Metro contract。
- 当前重点是关键站点与单实体追踪,不是完整全量 diff 矩阵。
- 大时间线或超大 payload 的优化暂未单独做 contract 收缩。

## 排查清单

遇到“看不到 / 跳不回 / 高亮不对 / 字段异常”时,按下面顺序排查:

1. 确认入口:
   - 是从 Current View 进入,还是从 Diff Studio Evidence Drawer 进入。
   - 如果是 Diff Studio 入口,确认是否真的点了“打开实体 Metro”。
2. 确认权限:
   - 当前账号是否仍有该表查看权限。
   - 如果字段显示为脱敏结果,先判断这是否符合当前角色预期,不要直接视为异常。
3. 确认上下文:
   - Diff Studio 入口是否带了相关字段和 `record_id`。
   - 如果某次 diff 没有明确匹配到版本,可能不会出现 `record_id` 高亮。
4. 确认返回链路:
   - 从 Diff Studio 进入时,应优先返回原来的 Diff Studio 页面和 query。
   - 如果缺少返回参数,回到 Diff Studio 默认页属于预期降级。
5. 确认数据语义:
   - 缺失字段、显式 `null`、终止记录在时间线里是不同语义,不要混为一谈。
   - 如果怀疑是权限或序列化边界问题,优先复核后端 timeline 测试结论。

## 何时上报

出现下面情况时建议上报给开发或管理员:

- Current View、Diff Studio、Entity Metro 对同一字段的权限表现不一致。
- 打开 Entity Metro 后无法回到原 Diff Studio 页面。
- 明明来自 Diff Studio,但字段高亮和 `record_id` 高亮始终缺失。
- 关键站点或全部版本模式与业务理解明显不符。
