# 审批人为空时新增数据提交审批处理说明

## 适用场景

用户在表格中新增或修改数据后，发现不能直接发布生效，而是需要点击“提交审批”。同时，审批人下拉框没有可选人员，导致草稿无法提交。

## 现象

- 新增数据后，数据先进入草稿批次。
- 草稿操作区显示“提交审批”，而不是“发布生效”。
- 系统要求选择审批人。
- 审批人下拉框为空，无法继续提交。

## 原因

这张表开启了“启用审批”。开启后，所有新增、修改、终止等变更都不会直接生效，而是需要先提交给审批人处理。

审批人为空通常是因为当前表没有可用审批人。系统中的审批人必须满足以下条件：

- 审批人必须是该表的 owner 或 editor 协作者。
- 审批人不能是当前草稿的创建者本人。
- viewer 协作者不能作为审批人。
- 被停用或离职的用户不会出现在审批人列表中。

因此，如果当前用户是表 owner，且表里没有其他 active editor，审批人列表就会为空。

## 用户处理步骤

### 方式一：这张表不需要审批

联系表 owner 或管理员，进入：

```text
表设置 -> 基本信息 -> 取消“启用审批” -> 保存
```

保存后，新产生的草稿可以按无需审批的方式发布生效。

如果当前已经有旧草稿仍显示“提交审批”，建议放弃旧草稿后重新新增数据。因为旧草稿可能保留了创建时的审批状态。

### 方式二：这张表确实需要审批

联系表 owner 或管理员，进入：

```text
表设置 -> 协作者与审批人 -> 添加协作者
```

添加一个可审批用户，并将角色设置为：

```text
editor
```

然后回到草稿详情页，重新选择审批人并提交审批。

### 方式三：不能放弃旧草稿

如果旧草稿里已经填写了较多内容，不方便放弃，可以先让管理员或 owner 添加一个 active editor 审批人，再继续提交当前草稿。

## 管理员检查清单

管理员或表 owner 可以按以下顺序检查：

1. 进入表设置，确认“启用审批”是否被勾选。
2. 如果业务不需要审批，取消勾选并保存。
3. 如果业务需要审批，进入“协作者与审批人”。
4. 确认至少有一个 active editor。
5. 确认该 editor 不是当前草稿创建者本人。
6. 让用户刷新页面或重新打开草稿详情，再选择审批人提交。

## 推荐给管理员的说明话术

```text
这张表可能开启了启用审批，但当前没有可用审批人。
请在表设置里取消启用审批，或在协作者与审批人里添加一个 active editor。
审批人不能是我本人，viewer 也不能作为审批人。
```

## 相关规则

- 开启审批后，草稿发布按钮会变为“提交审批”。
- 未开启审批时，草稿可以直接“发布生效”。
- 审批人只能从表 owner 和 active editor 中选择。
- 草稿创建人不能审批自己的草稿。
- 当前草稿是否需要审批，会在提交时根据表的最新审批设置重新判断。
